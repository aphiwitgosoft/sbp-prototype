หลักการออกแบบ (มาตรฐานสากล): เก็บที่ตาราง system_configs แบบ key–value · config_key ใช้ dot notation <หมวด>.<ชื่อค่า> · value_type (NUMBER / STRING / BOOLEAN / JSON / CRON) ใช้ validate ค่าก่อนบันทึก · is_editable = false คือค่าคงที่ทางธุรกิจ (เช่น รัศมี 1/2 กม. · วงเงิน 100,000 · เกณฑ์ 60 วัน · เกณฑ์ −10 ตามข้อ 8.2 ของเอกสาร Batch) — แก้ผ่านหน้าจอ/API ไม่ได้ ต้องขออนุมัติและ deploy · ทุก service อ่านผ่าน API GET /configs/{key} พร้อม cache 5 นาที และถูก invalidate เมื่อมีการแก้ไข · ห้ามเก็บ secret (รหัสผ่าน / API key / connection string) — ต้องอยู่ Secret Manager เท่านั้น (ข้อปรับปรุง P0) · พารามิเตอร์เฉพาะราย job อยู่ที่ job_configs (หน้า Batch Job) — ตารางนี้เก็บเฉพาะค่ากลางที่ใช้ร่วมทั้งระบบ · ทุกการเปลี่ยนแปลงบันทึกประวัติลง audit_logs พร้อมเหตุผล
