3. System Context and End-to-End Flow
3.1 System Context
   QSSI SFTP provides monthly score files; Job 1 maps them into FCS_QSSI_SCORE.
   ALLMAP SQL Server provides impact-store pairs and competitor records for Jobs 2 and 3.
   IAS/MIS exchanges AMS06001O outbound and AMS06001I inbound sales files for Jobs 4 and 5.
   BPM receives three independent SFTP files from Jobs 7-9; K2 workflow is started separately by Job 8b.
   STA receives FRBC0001 from Job 6; Job 10 monitors overdue ACK only for STA compensate datasets.

Inputs and Preconditions
   Remote directory /export/qssishare/onl/qssi/textfile/SBP/QSSI_Monthly/
   SFTP port 218; four prefixes mrs1trnf_, mrs2trnf_, mrs3trnf_, mrs5trnf_
   Local input /appshare/SPS/FCS/interface_data/in/; WINDOWS-874; batch size 10,000

Business Rules and Implementation Notes
   Filename uses the current month, while the DB period written is the previous month.
   RS1 mapping is stateful: D+E, F+G+H, I+J, K+L; H also emits category 16.
   Backup move runs even when import fails, so there is no automatic retry.
   Temp table is deleted per file and must not be processed concurrently.

Output, Error Handling and Re-run
   Output FCS_QSSI_SCORE; data later feeds Job 6 QSSI categories 8,9,12,1,10,16.
   On exception, rollback to savepoint and send failure email; source file still moves to backup.

7. Operational Runbook
7.1 Pre-Run Checklist
        Confirm business date, scheduler window and active datasource/environment.
        Confirm upstream file or ALLMAP view is available and has expected period.
        Check previous run status, live output folder, backup folder and FGI_CONFIRM_RECEIVE_DATA.

        For Job 6, reconcile all 10 state mutations before confirming successful business output.
