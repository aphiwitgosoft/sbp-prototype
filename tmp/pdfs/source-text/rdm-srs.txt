1. SRS Overview
          เอกสารนี ้ใ ช้ อธิ บ ายระบบประกั น รายได้ ทางฝั่ ง K2 โดยจะอธิ บ ายในส่ วนของ Functional
   Requirement ซึ่งเป็ นส่ วนที่มีค วามส าคัญประกอบด้ วย Module ต่ างๆ ที่ จาเป็ นต่ อการใช้ งานเช่น
   หน้ าจอ Landing Page, หน้ าจอส าหรั บ จั ด การข้ อมู ล Master, หน้ าจอส าหรั บ ด าเนิ น การ
   (โดยหน้ าจอจริ งอาจมีก ารเปลี่ ย นแปลงบ้ าง ตามความเหมาะสม) รวมไปถึ ง Flow การด าเนิ นการ
   และรูปแบบรายงานต่าง ๆ เป็ นต้ น

       3.2 Non-Functional Requirement

                3.2.1 Performance Requirements
                     • รองรับจานวนผู้ใช้ งานพร้ อมกันโดยเฉลี่ย 80 คน และสูงสุด 100 คน
                     • ตอบสนองต่อผู้ใช้ งาน 30 วินาที ระหว่างผู้ใช้ งาน ที่สภาวะแวดล้ อม Network และ Hardware ปกติ
                3.2.2 Reliability
                     • ระบบต้ องบันทึกข้ อมูลทุก Transaction ใน Process ที่แจ้ ง User ว่า Complete
                     • ระบบต้ องไม่บันทึกข้ อมูลแต่ละ Transaction ที่ทาไปแล้ วใน Process ที่แจ้ ง User ว่า error
                3.2.3 Usability
                     • ผู้ใช้ งานระบบสามารถใช้ งานได้ ง่ายโดยผ่าน Web Browser ของอุปกรณ์ที่ใช้ ในการเชื่อมต่อ Internet
