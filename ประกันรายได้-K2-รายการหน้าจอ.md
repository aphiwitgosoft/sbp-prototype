# รายการหน้าจอทั้งหมด — ระบบประกันรายได้ (K2)

> อ้างอิงจากเอกสาร Software Requirement Specification "ประกันรายได้" Version 3.1 (Gosoft Thailand)

ระบบประกันรายได้พัฒนาขึ้นเพื่อชดเชยรายได้ให้ร้าน 7-11 Franchise ที่ได้รับผลกระทบเมื่อมีร้าน 7-11 ใหม่เปิดในรัศมีใกล้เคียง โดยรับข้อมูลจากทีม Store Chain Unit (text file รอบ 17:00 ทุกวัน) และเชื่อมต่อกับระบบ Finance & Account Unit เป็น Web Application ใช้งานผ่าน Google Chrome

---

## หน้าจอหลัก (Main Functions)

### 1. หน้ารอท่านดำเนินการ (3.1.2)
แสดงเฉพาะเอกสารที่ผู้ใช้งานคนนั้นๆ ถูกกำหนดสิทธิ์ให้ดำเนินงานเท่านั้น แสดงผลเป็นตาราง 9 คอลัมน์:

- ครั้งที่
- เลขที่เอกสาร
- รหัสร้านถูกกระทบ
- ชื่อร้านถูกกระทบ
- ภาค
- ยอดขายที่ลดลง
- จำนวนเงินที่ชดเชย
- สถานะ
- รอดำเนินการ (วัน)

คลิกที่รายการเพื่อเปิดเอกสารนั้นๆ

**หมายเหตุพิเศษ:** หากร้านถูกกระทบเป็นร้านที่ข้อมูลยอดขายไม่ครบ 60 วัน ระบบจะแสดงข้อความ "ผิดปกติ" สีแดงที่ด้านหน้ารายการ และตัวหนังสือทั้งแถวจะแสดงเป็นสีแดง

**Actor:** ผู้ใช้งานทั่วไป

---

### 2. หน้าเอกสารที่เกี่ยวข้องกับท่าน (3.1.3)
แสดงทุกงานที่เกี่ยวข้องกับบุคคลนั้นๆ (จะดำเนินการได้ก็ต่อเมื่ออยู่ในสถานะของบุคคลนั้น) โครงสร้างตาราง 9 คอลัมน์เหมือนหน้ารอท่านดำเนินการ แต่ขอบเขตข้อมูลกว้างกว่า ครอบคลุมเอกสารที่เคยเกี่ยวข้องทั้งหมด ไม่จำกัดเฉพาะที่รอดำเนินการ

**Actor:** ผู้ใช้งานทั่วไป

---

### 3. หน้าเอกสารข้อมูลร้านถูกกระทบ (3.1.6) — หน้าจอหลักที่ซับซ้อนที่สุด
หน้าจอดำเนินการตาม workflow ประกอบด้วยหลายส่วน (ดูรายละเอียดในหัวข้อ "ส่วนย่อย" ด้านล่าง)

- เลขที่เอกสารรูปแบบ `YYYY/Running 5 Digit` (YYYY = ปี ค.ศ. ที่สร้างเอกสาร, running เริ่มต้นที่ 00001)
- หน้าจอแสดงผลแตกต่างกันตาม role/step ของผู้ดำเนินการ
- ส่วนที่แก้ไขได้ขึ้นอยู่กับขั้นตอน ที่เหลือเป็น Read Only

**Actor:** ผู้มีสิทธิ์ดำเนินการในเอกสารข้อมูลร้านถูกกระทบ

---

### 4. หน้ากำหนดชื่อผู้ปฏิบัติงาน (3.1.8)
กำหนดว่าใครรับผิดชอบตำแหน่งใดในระบบ

- **ชื่อพนักงาน** (required) — กดแว่นขยายเลือกจาก Pop Up
- **ชื่อตำแหน่ง** (required) — เลือกจาก Drop Down List
- **ภาค** — แสดงเฉพาะเมื่อเลือกตำแหน่ง "ส่งเสริมธุรกิจพันธมิตรฯ"

ปุ่ม: เพิ่มข้อมูล / เคลียร์ค่าเริ่มใหม่ / แก้ไขข้อมูล (ต้องระบุเหตุผลการแก้ไข)

บันทึกลงตาราง `MaintainMasterHistory` และ `CompenOrganizeProfile`

**Actor:** Role Admin, HQ และ User Admin

---

### 5. หน้ากำหนดปัจจัยภายนอก (3.1.9)
จัดการ Master ข้อมูลปัจจัยภายนอก

- **รหัสปัจจัยภายนอก** (required) — ห้ามซ้ำ
- **ชื่อปัจจัยภายนอก** (required)
- **รายละเอียดเพิ่มเติม**

ปุ่ม: เพิ่มข้อมูล / เคลียร์ค่าเริ่มใหม่ / แก้ไขข้อมูล (ต้องระบุเหตุผลการแก้ไข)

บันทึกลงตาราง `FactorProfile` และ `MaintainMasterHistory`

**Actor:** Role Admin, HQ และ User Admin

---

### 6. หน้ารายงานสรุปสถานะ (3.1.7)
ออกรายงานตามเงื่อนไขค้นหา

**เงื่อนไขค้นหา:** รหัสร้านถูกกระทบ, ชื่อร้าน (auto), เดือน/ปี (ถึง), ประเภทร้าน (FR Type A/B/C/พนักงาน เลือกได้มากกว่า 1), ภาค (BE/BN/BS/BW/RC/RE/RN/RS เลือกได้มากกว่า 1), สถานะ (ทีละ 1), Period Statement From–To

**ปุ่ม:** ค้นหาข้อมูล (ต้องระบุปี, ออกเฉพาะที่มีเลขที่เอกสาร) / เคลียร์ค่าเริ่มใหม่ / Export Excel

**ผลลัพธ์ 19 คอลัมน์:** รหัสร้านถูกกระทบ, ชื่อร้านถูกกระทบ, ภาค, ประเภทร้าน, เดือน/ปีที่ชดเชย, วันที่โอนเป็นร้าน SP, Period Statement, รหัสร้านเปิดใหม่, ชื่อร้านเปิดใหม่, ภาค, ประเภทร้าน, ยอดเงินชดเชย, สถานะ, ชื่อ-นามสกุลผู้ดำเนินการ, ผลการพิจารณา, รอดำเนินการ(วัน), ครั้งที่, วันที่สร้าง, เลขที่เอกสาร

**Actor:** Role Admin, HQ และ Report Admin

## ส่วนย่อยภายในหน้าเอกสารข้อมูลร้านถูกกระทบ

หน้านี้แสดงแตกต่างกันตาม role/step ของผู้ดำเนินการ ส่วนที่แก้ไขได้ขึ้นกับขั้นตอน ที่เหลือเป็น Read Only

| ส่วน (View) | ทำอะไร |
|---|---|
| **Create New Document** | สร้างเอกสารร้าน Store Partner ที่อยู่นอกเหนือเงื่อนไขของระบบ |
| **สร้างเอกสารที่ FS** | สร้างผ่านระบบ Finance & Account Unit รอ SBP Statement ประมวลผลส่งมา ~1 วัน (เฉพาะผู้มีสิทธิ์เข้าถึง) |
| **ข้อมูลร้านถูกกระทบ** | แสดงข้อมูล 16 ฟิลด์ (รอบ/ครั้ง/เดือน, สถานะ, เลขที่, วันที่สร้าง, ผู้ดำเนินการ, รหัส/ชื่อร้าน, ภาค, ประเภทร้าน, เจ้าของร้าน, นิติบุคคล, วันที่โอน, ข้อมูลยอดขายเพิ่มเติม, จำนวนเงินชดเชย, ยอดขายที่ลดลง, เอกสารแนบ PDF) — **Read Only** |
| **ร้านเปิดใหม่** | ข้อมูลร้าน 7-11 ที่เปิดใหม่ทำให้กระทบ (รหัส, ชื่อ, ภาค, ประเภทร้าน 8 ชนิด, เจ้าของ, นิติบุคคล, วันเปิด/ปิด, ระยะห่างตามจริง, %ชดเชย, เงินชดเชย) — แก้ไข %ชดเชยได้เฉพาะบางตำแหน่ง |
| **แผนที่** | ฝัง Link จาก ALLMAP แสดง POI ร้านเปิดใหม่/ถูกกระทบ/คู่แข่ง พร้อมรัศมี 1 กม.(กทม.+ปริมณฑล) / 2 กม.(ต่างจังหวัด) มีปุ่ม Link To ALLMAP |
| **ประวัติการชดเชย** | ครั้ง, เดือน/ปีที่กระทบ, จำนวนเงินที่ชดเชย, เดือน/ปีที่ส่งบัญชี (วงเงิน ≤100,000 = GM Operation / >100,000 = AVP Operation), สถานะ, ผลพิจารณา, เปิดลิงค์เอกสาร |
| **ผลการพิจารณา** | ชื่อผู้พิจารณา, ตำแหน่ง, ผลการพิจารณา, รายละเอียดการพิจารณา, วัน/เวลา |
| **ร้านคู่แข่งเปิดกระทบ** | รายชื่อคู่แข่งจาก Master (108 Shop, V Shop, Lotus Express, AM PM, Joy, Max Valu, Lawson 108, CJ Express, Tops Daily, Family Mart ฯลฯ), วันที่เปิด, รายละเอียด — บาง role เพิ่ม/แก้ไข/ลบได้ |
| **ปัจจัยอื่นๆ** | ปัจจัยภายนอก (dropdown), วันที่เริ่มต้น, วันที่สิ้นสุด (validate สิ้นสุด ≥ เริ่มต้น), รายละเอียดเพิ่มเติม |
| **เอกสารแนบทั้งหมด** | ไฟล์แนบ, ตำแหน่ง, ผู้สร้างแนบไฟล์, รายละเอียด, วัน/เดือน/ปี |
| **คำนวณเงินชดเชย** | คำนวณผ่าน Finance & Account Unit (เฉพาะเจ้าหน้าที่ SBP DSA) |
| **พิจารณา** | ตัวเลือกส่งงานต่อ (ต่างกันตาม role), ความคิดเห็นเพิ่มเติม, แนบไฟล์ (≤5 MB), ปุ่มส่งดำเนินการ + ปุ่มบันทึก |

**ประเภทไฟล์แนบที่รองรับ:** vsd, dwg, afp, pdf, mda, zip, wav, mp3, gif, jpg, tif, tiff, htm, html, txt, xml, mpg, mov, ivs, doc, docx, xls, xlsx, pps, ppt, pot, csv

## ลำดับขั้นตอนผู้ดำเนินการ (Workflow Steps) ในส่วนพิจารณา

หน้าเอกสารเดียวกัน แต่ตัวเลือก "พิจารณา" และสิทธิ์แก้ไขเปลี่ยนตามตำแหน่งที่เอกสารวิ่งไปถึง:

| ลำดับ | ตำแหน่ง | SectionCode | ตัวเลือกส่งงาน / หมายเหตุ |
|---|---|---|---|
| 1 | **ฝ่าย SBP DSA** (Manager Franchise) | 06 | ไม่ชดเชย / หยุดชดเชย / ส่งเสริมธุรกิจ SBP / บัญชี SBP / เจ้าหน้าที่ SBP DSA |
| 2 | **เจ้าหน้าที่ SBP DSA** (Officer Franchise) | 08 | มี view คำนวณเงินชดเชยเพิ่ม |
| 3 | **ฝ่ายส่งเสริมธุรกิจฯ** (Manager OPT) | 01 | แก้ไขร้านเปิดใหม่ + คู่แข่ง + ปัจจัยอื่นได้ |
| 4 | **GM ส่งเสริมธุรกิจฯ** (GM OPT) | 02 | เงิน >100,000 → AVP / <100,000 → บัญชี/DSA |
| 5 | **ผู้บริหารสำนักบริหาร SBP** (AVP OPT) | 03 | เห็นควรชดเชย / ไม่ชดเชย / ส่งกลับ GM |
| 6 | **ฝ่ายบัญชี SBP** (Manager Account Franchise) | 04 | บัญชีปฏิบัติการภาค / ฝ่าย SBP DSA |
| 7 | **บัญชีปฏิบัติการภาค** (Operation Account) | 05 | บัญชีภาคอนุมัติ = **จบ workflow** |

**Validation ปุ่มส่งดำเนินการ:** ต้องเลือกผลการพิจารณาอย่างน้อย 1 ข้อ มิฉะนั้นแสดง pop up "ท่านยังไม่เลือกผลการพิจารณา กรุณาเลือกข้อมูลก่อนกดส่งดำเนินการ" (กรณีเลือกไม่ชดเชย/ฝ่ายบัญชี SBP — ความคิดเห็นเป็น required field)

**ปุ่มบันทึก:** บันทึกข้อมูลโดยไม่มีการ validate ใดๆ

## สิทธิ์การเข้าถึงเอกสาร (3.1.1)

ตารางกำหนดสิทธิ์เมนูตาม role 8 กลุ่ม:

| Code | Role | คำอธิบาย |
|---|---|---|
| 00 | Default | กลุ่มผู้ดำเนินการในแบบฟอร์ม |
| 01 | Admin | ผู้ดูแลระบบ BPM เห็นทุกเมนู จัดการได้ทุกอย่าง |
| 02 | HQ | สำหรับ HQ Support |
| 03 | User Admin | ให้ User เป็น Admin ระบบ |
| 04 | Report Admin | ดูเมนูรายงาน/รายงานสรุป |
| 05 | Assign Job | แจกงานที่เมนูข้อมูลผิดปกติ |
| 06 | Report Admin Special | เรียกดูเอกสารทั้งหมด |
| 10 | UserViewer | รายชื่อผู้มีสิทธิ์ดูเอกสาร (Role 01, 02 บริหารจัดการ) |

ครอบคลุมเมนู: ALLProcess, หน้าแรก, สถานะบันทึก, Help, รายงาน, ผู้ดูแลระบบ และเมนู Master Form (กำหนดสิทธิ์, ผู้ดำเนินการ, ปัจจัยภายนอก, ส่งอีเมล, URL, Migrate File S3 ฯลฯ)
