ผังกระบวนการ FGI Impact Store Compensation Flow

กระบวนการชดเชยรายได้สาขาที่ได้รับผลกระทบ แบบ End-to-End · AllMap → SBPGI → MIS → STA → SAP → K2 (BPM)

การไหลของข้อมูลหลัก (Main flow)
Callback / return (ทางกลับ)
ข้อมูลกำลังประมวลผล (live)

รายละเอียดแต่ละขั้นตอน (6 Phases)

1
รับข้อมูลผลกระทบ
Impact Data Intake · AllMap → SBPGI
  • AllMap ส่งข้อมูลร้านที่ได้รับผลกระทบไปยัง SBPGI
  • (Optional) นำเข้า/เตรียมข้อมูลใน SBPGI เพื่อใช้ประมวลผล
  • กรณีร้านถูกกระทบแต่ไม่มีข้อมูลสาขา ใช้ API พิเศษ updateCompensateFromFS
/fgiService/updateCompensateFromFS
2
ส่งออกและประมวลผลที่ MIS
MIS Export & Processing
  • Job FGI_ExportImpactStoreToAMS รันเวลา 16:00 (วันที่ 7–16 ของเดือน)
  • MIS อ่านไฟล์และประมวลผลข้อมูลร้านที่ได้รับผลกระทบ
16:00AMS060010_AMS060011_
3
ส่งออกและตรวจสอบที่ STA
STA Export & Validation
  • Job FGI_ImportImpactStoreSale.sh รัน 16:30 นำเข้ายอดขายจาก MIS
  • Job FGI_ExportImpactStoreToSTA.sh รัน 17:00 ทุกวัน
  • STA รับไฟล์และประมวลผล (FRBC0001_)
16:30 / 17:00FRBC0001_
4
กระบวนการอนุมัติที่ STA
Approval Decision · A / S / I
  • Approve (A) → ตรวจสอบ + บันทึกบัญชี (Accounting Entry) ส่งไป SAP
  • Stop Flow (S) → หยุดกระบวนการ (End)
  • Initial (I) → ดำเนินการเริ่มต้นค่าชดเชย
Approve = AStop = SInitial = I
5
เริ่มต้นค่าชดเชยจาก FS
Initial Compensation Update from FS
  • STA วางไฟล์ยอดชดเชย แล้วเรียก API updateCompensateFromFS
  • ประเภทค่าชดเชย = Z, เงื่อนไขค่าชดเชย = W
  • SBPGI อัปเดตข้อมูลและตอบ return Success
RT040035_RT040078_updateCompensateFromFS
6
ยืนยัน K2 และปรับปรุงค่าชดเชย
K2 Confirmation & Adjust
  • (K2) ดึงข้อมูลจาก BPM และบันทึกสถานะการยืนยัน
  • 6.1 (Optional) Adjust Flow: ส่งข้อมูล Adjust เฉพาะร้านที่กระทบ แล้วอัปเดต
/fgiService/confirmDataFromBPM
