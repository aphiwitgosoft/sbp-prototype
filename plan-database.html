3. กุญแจเชื่อมข้ามระบบ (Cross-System Keys)

  • impacted_stores.store_code = fgi_impact_stores.impacted_store_code — สะพานหลักระหว่างโซน C (K2) กับโซน A (FGI/FCS) ใช้รหัสร้าน 5 หลักเดียวกันทั้งระบบ ทั้งสอง
  • *.impact_process_id → fgi_impact_processes.id — hub กลางของคู่ร้าน ยอดขาย และคู่แข่งในหนึ่งรอบชดเชย ลดการ join ด้วย business key หลายคอลัมน์ ใหม่
  • compensation_documents.impact_process_id → fgi_impact_processes — FK ใหม่ 1 รอบชดเชย : 1 เอกสาร แทนการส่งไฟล์ BPM06001O (48 ฟิลด์) ข้ามระบบ ใหม่
  • workflow_instances.doc_no → compensation_documents — เปิด instance เมื่อผ่าน Gen Flow Gate · สถานะ instance ทำหน้าที่แทน workflow_generation_status = Y ของเดิม ใหม่
  • document_competitors.source_system = 'ALM' — แถวที่มาจาก fgi_impact_competitors (Jobs 3/7 เดิม) แยกจากที่ผู้ใช้เพิ่มเอง (USER) FGI/FCS
  • compensation_histories.submit_account_month — งวดที่ส่งเข้าไฟล์ FRBC0001 ไป STA (Job 6) · สถานะ I/C/A/N/S/Z ตามเดิม FGI/FCS
  • interface_transactions — คอลัมน์ FK แยกประเภท (impact_process_id / sales_summary_id / doc_no) + data_name เป็น enum — เลิก parseInt(impacted_store_code) ที่ทำเลขศูนย์นำหน้าหาย ใหม่

4. ข้อปรับปรุงจากระบบเดิม

P0 × 3 · P1 × 4
  1. เลิก polymorphic FK — transaction_key ของ tracking เดิมชี้คนละตารางตาม data_name (P1) → interface_transactions ใช้ typed FK แยกคอลัมน์
  2. บังคับ status domain ด้วย enum / check constraint — W/P/Y/N · I/C/A/N/S/Z · action_status (Y/W/N) · workflow_generation_status (W/Y/N) ตามที่เอกสารแนะนำ
  3. ครอบ Job 4 ด้วย transaction (outbox pattern) — เดิม commit W→P ก่อนเขียนไฟล์ rollback ไม่ได้ (P0 อันดับหนึ่งของระบบ)
  4. แก้บั๊ก purge tracking (E20) — SQL เดิมต่อ data_name สองค่าเป็น string เดียว ทำให้ไม่เคยลบ — ต้องทำพร้อม data migration และ test
  5. ทบทวน NULL → auto-accept ของ Job 5 (P1) — ระบบใหม่ตั้งสถานะ "รอตรวจสอบ" แทนการ accept อัตโนมัติ · ต้องขอ business sign-off ก่อนเปลี่ยน
  6. ย้าย credential ทั้งหมดไป Secret Manager + บังคับ TLS — เดิม SFTP/K2 Basic Auth เป็น plaintext/hardcoded (P0)
  7. ใช้ identity ต่อตารางแทน sequence รวม 7 ตัว — คงชื่อ sequence เดิมไว้เฉพาะช่วง migrate (Errata E18)
  8. golden-file tests ต่อ interface ภายนอก — encoding WINDOWS-874 / UTF-8 / TIS-620, วันที่ พ.ศ., ชื่อประกอบ first + last — กันพฤติกรรมเพี้ยนตอน re-implement
