1. สถาปัตยกรรมระบบใหม่ — แยก Frontend / Backend

BE / FE Split

จุดเปลี่ยนสำคัญ: ไม่มี BPM/K2 engine ภายนอกอีกต่อไป — การเปิดและเดิน workflow ทำโดย Workflow Engine ใน Backend เอง (แทนไฟล์ BPM06001O/2O/3O + K2 REST StartInstance ของ Job 8/8b/9) ส่วน interface กับระบบภายนอก (QSSI, ALLMAP, IAS, STA) คงกลไกไฟล์/SFTP เดิมไว้เพราะเป็นระบบของทีมอื่น

สัญญากลางที่ผูกกับ workflow: ดู LLDD-BE-API-Common-Contracts และ LLDD-FE-Integration-Contracts ก่อน implement ทุกจุดที่เรียก workflow API — POST /documents/{docNo}/actions รับ {result, comment} และคืน {nextSection, statusCode, status}; POST /workflows/instances ใช้ service token และ Gen Flow Gate W/Y/N อยู่ที่ BE Workflow Engine

4. พฤติกรรมจาก Approve Flow เดิมที่รับเข้าระบบใหม่

Flow ประกันรายได้.png

จากแผนภาพอนุมัติระบบเดิม (BPMN "A-Document Approve Flow ประกันรายได้") มี 4 พฤติกรรมที่ flow ด้านบนยังไม่ครอบคลุม — ทั้งหมดอยู่ฝั่งเอกสาร/Workflow (โซน B/C) ไม่กระทบ FGI/FCS pipeline (Jobs 1–10) และ interface ภายนอก

พฤติกรรมจาก flow เดิมระบบใหม่ขอบเขตผลกระทบ
อีเมลเตือนงานค้างรายสัปดาห์ ส่งอีเมลแจ้งเตือนผู้ดำเนินการ ทุกวันจันทร์ 10:00 แทบทุกขั้น (จุด 10.1, 20.2, 30.1, 70.1, 80.1, 110.2) Notification Service เพิ่ม reminder job รายสัปดาห์ อ่านงานค้างจาก workflow_tasks · ผู้รับตาม status_email_rules · รอบเวลาแก้ได้ใน config โซน B/C · ไม่แตะ FGI/FCS
Escalation งานค้าง 30/45/60 วัน ส่งต่อไปยัง GM OPT เมื่องานค้างครบ 30, 45, 60 วัน (จุด 20.3) Workflow Engine ตรวจ waiting_days ของ workflow_tasks แล้วแจ้ง/ส่งต่อหัวหน้า Section ตามเกณฑ์ 30/45/60 วัน (ค่ากำหนดแก้ได้) โซน B · ไม่แตะ FGI/FCS
สร้างเอกสารแบบ manual ขั้น 15 (lane admin): กรอกข้อมูลร้านเปิดใหม่/ร้านถูกกระทบเอง เมื่อระบบคำนวณให้ไม่ได้ หน้า "สร้างเอกสาร" (k2-create) → POST /documents ลงตารางชุดเดียวกับเส้นอัตโนมัติ — เป็นจุดเข้าเสริมฝั่งผู้ใช้ ไม่แตะ batch โซน B · ไม่แตะ FGI/FCS
เงื่อนไขร้านก่อน/หลัง 1/10/2557 โน้ต 10.1–10.4 แยกจุดเริ่มเอกสารตามวันที่โอนร้านแฟรนไชส์ และกรณีไม่มีข้อมูลให้คำนวณ ถ้ายืนยันตาม SRS: ใช้เป็นกฎ routing ตอนเปิด instance ใน Workflow Engine — เกณฑ์ Gen Flow Gate ฝั่ง batch คงเดิมทุกข้อ · รอ verify กับ SRS v3.1 โซน B · ไม่แตะ FGI/FCS

กติกาธุรกิจทั้งหมดคงเดิมตามเอกสารอ้างอิง — เกณฑ์ Gen Flow Gate, กฎวงเงิน 100,000, หน้าต่างคำนวณ 4×15 วัน, เกณฑ์ outlier ≥ 50, QSSI 6 หมวด, %ชดเชยรวม 100% ฯลฯ การเปลี่ยนแปลงจำกัดอยู่ที่กลไกทางเทคนิค (ไฟล์ภายใน → DB/API, K2 engine → Workflow ภายใน) และการแก้ความเสี่ยง P0/P1 ที่เอกสาร v4.0 ระบุ — การเปลี่ยนพฤติกรรมเชิงธุรกิจใด ๆ ต้องขออนุมัติแยกตามข้อ 8.2 ของเอกสาร
